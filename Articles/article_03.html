What Is Useful 

When a scene contains a large number of curves with irregular order and naming, distributed across different groups, manually selecting Base curves that are at the same or nearby positions to match Target curves and create BlendShapes is nearly impossible.

Search And BlendShape For Curves provides intelligent automatic matching. You can also define a maximum search radius in world space for the first CV (Base CV) of the Target curves relative to the Base curves. The tool will use this threshold to automatically match curves and create the BlendShapes.

In addition, you can select different numbers of Base and Target curves. For example, if you select 5 Base curves and 10 Target curves, the tool will automatically find, for each Base curve, the closest curve among the 10 Target curves as its match. Conversely, it can also handle cases where the number of Base curves exceeds the number of Target curves, intelligently performing the matching.

If the Base and Target curves have inconsistent topology and the BlendShape cannot be created, you can enable the “Rebuild Curve” topology option to force the tool to reconstruct the target curves, ensuring successful BlendShape creation. Each created Base curve is automatically added to a set named “base_curve_set*n” for easy viewing and management.

Regardless of any exceptions encountered during execution, the tool will display a prompt window asking whether to proceed, providing a safe and convenient workflow for creating BlendShapes.



Search And BlendShape For Curves Options
_____________________________________________________________________________________________________


Base Curves

Select the transform nodes or shape nodes of the NURBS curves that you want to deform — these will be the “Base Curves” for the BlendShape. Then click the “Base Curves” button.


Target Curves
 
Select the transform nodes or shape nodes of the target NURBS curves. Each Target represents a possible pose or deformation that the Base curve will achieve. Then click the “Target Curves” button.


P1 :

You can select different numbers of Base curves and Target curves (the number of Base curves can be greater than, equal to, or less than the number of Target curves). For example, if you select 5 Base curves and 10 Target curves, the tool will automatically find, for each Base curve, the closest curve among the 10 Target curves as its match.


Search radius value (Option Menu)

With the ‘is calculated automatically’ option: There will be no restriction on the distances between the Base CV positions of the base curves and target curves in the scene; it will simply match the nearest curve.

With the ‘is specified manually’ option: Enables the float field next to it, allowing you to define a specific distance threshold for matching. This provides granular control, useful in dense scenes or when specific matching criteria are required.


BlendShape node number

With option ‘one’: Consolidates all matched target shapes into a single BlendShape node, where each target curve becomes a target in that one node. This can lead to a cleaner Outliner if you prefer fewer nodes.

With option ‘multi’: Creates separate BlendShape nodes for different matches or groups of matches. Depending on your rigging conventions or the complexity of your setup, this might offer more modular control or prevent single BlendShape nodes from becoming overly complex.


BlendShape envelope controlled
 
This section controls how the resulting BlendShape's envelope (weight) will be driven.

Option ‘by default’: An attribute named ‘Target Curve Envelope’ will be created on set of the ‘base_curve_set*n’. in Maya Outliner. This attribute controls the BlendShape’s Envelope, as shown in the image below.




Option ‘by control attribute’: Enables the sub-section below to link the envelope to an external controller. This is ideal for hierarchical control or global on/off switches.


Control Attribute Sub-Section
 
(Only visible if "by control attribute" is selected above)

Select Controller: Select the object you want to use as the driver in the Maya viewport (e.g., a custom locator, a control curve) and click the ‘Select’ button. The selected object's name will appear in the text field.


Attribute Dropdown: Once a controller is loaded, this dropdown populates with existing float/double attributes on that controller that are marked as keyable.

Custom Attribute: Select "Custom Attribute" from the dropdown to create a new double (float) attribute on the selected controller. A prompt dialog will appear asking for the new attribute's name. This new attribute will then be used to drive the BlendShape envelopes.


Rebuild the target curve if its topology isn't the same as the base curve
 
BlendShapes in Maya strictly require that the base and target objects have the exact same number of control points (CVs) and topological structure. If there's a mismatch, the BlendShape operation will fail.

If checked, this critical feature addresses topology discrepancies. The tool will intelligently attempt to rebuild target curves to match the number of CVs and spans of their corresponding base curves before creating the BlendShape. This helps prevent errors and ensures a successful BlendShape creation even if your target shapes were created with slightly different curve parameters.


Menu Bar
 
Edit > Save Settings: Persists your current UI selections and values to a preset file (ui_searchAndBlendShapeForCurves_Presets.py) located in your Maya script path. The next time you open the tool, it will load these saved settings, allowing for a consistent workflow across sessions.

Edit > Reset Settings: Reverts all UI options and fields to their factory default values. This is useful for starting fresh or troubleshooting. It also removes any previously saved preset file from the file directory.

Tip: File directory: /home/user/maya/scripts/ui_searchAndBlendShapeForCurves_Presets.py


Help > Help on 'Search And BlendShape For Curves': Opens an external PDF help document (linked online) providing more in-depth usage examples and visuals.

Help > Information: Displays tool metadata, including author, version, Python version, and creation/update dates.












Notification Dialog Window
_____________________________________________________________________________________________________


Invalid Selection

If nothing is selected or the selection does not contain NURBS curves, a notification window will appear with the message “These selected objects are NOT NURBS curves” or “No NURBS curves detected in your selection,” along with the list: invalidList

‘Please select only NURBS curve transform nodes or NURBS curve shape nodes.’




No Overlap
 
A curve cannot be designated as both a Base curve and a Target curve. The tool will return a list of overlapping curves.




Same Curve Names in the Scene
 
If there are curves with the same name in the scene, a notification window titled 'Same Curve Name in the Scene' will appear with the dictionary: sameCurveNameDic

You need to click ‘Confirm’ to terminate the operation and return to the scene to rename those curves.




Two Or More Curves Have Same Base CV Position
 
You will receive a notification window when two or more curves share the same Base CV point position with the list: sameBaseCVPositionList

The notification window indicates that the Base CV position of ‘curve10’ is equidistant to the Base CV positions of ‘curve5’ and ‘curve12’. If you click ‘Continue’, the tool will forcibly assign the corresponding curves in order. Clicking ‘Cancel’ will abort the execution.




Invalid Closest Distance
 
If the ‘Search radius value’ is set to ‘is calculated automatically’, there will be no restriction on the distances between the Base CV positions of the base curves and target curves in the scene; it will simply match the nearest curve.

If the ‘Search radius value’ is set to ‘is specified manually’ and the specified value is too small, a notification window will appear when the distances between the Base CV positions of the base curves and target curves in the scene exceed this value. A dictionary of curves exceeding the limit, hasTooLargeDistanceDic, will be returned.





The notification window indicates that the distance between Base Curve ‘curve 5’ and Target Curve ‘curve 13’ exceeds the specified radius of 0.4.

Clicking ‘Continue’ will force the tool to ignore curves that exceed the distance limit.


Topology Mismatch
 
If the corresponding Base curve and Target curve have different topologies, a BlendShape cannot be created. In this case, you need to enable ‘Rebuild the target curve if its topology isn’t the same as the Base curve’ in the UI. This will rebuild the Target curve’s topology to match that of the corresponding Base curve.

The invalidTopologyCurveDic will be displayed in the notification window.




The Number Of Base Curves And Target Curves Is Different
 
If the number of Base curves and Target curves is different, a notification window will appear showing the respective counts. The user can choose to use the number of Base curves as the reference, which means that it will select a matching number of Target curves (ignoring any extra Target curves); the reverse is also possible.

Clicking ‘Stop Process’ will stop the execution of the process.
